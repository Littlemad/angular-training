Main Landing

Welcome to my personal sandbox for all things Angular. I use this space to document coding patterns, test out experimental ideas, and build a library of tools and components that I can reference in my future projects.

I try to structure the code in a way that follows few code methodologies and some personal preferences:

  • Following SOLID principles
  • Using modern naming convention for angular files without the definition of the "-component-" suffix (and isntead identify the component by their use dividing pages, component and services to be trackable by namefiles)
  • Learning about architecture and organizing to better maintainibility
  • Architecturally for SASS, I am using the ITCSS methodology
  • Defining class names approaching BEM
